Function Forms conversion

Throughout the course will be a mix of ES 6 and ES 5, but it mainly the same below is a conversion of writing a function.

ES 5

Here we use the basic function call

function compressAllBoxes(boxes) {
   boxes.forEach(function(item) {
    console.log(item);
     });
 }

Main differences:

  • Usually has the word function typically used on the outermost Function
  • use of the word function a lot more wordier

ES 6

Here we use the basic function call the more modern way

const compressAllBoxes = (boxes) => {
   boxes.forEach(boxes) => console.log(boxes));
     };

Main differences:

  • replace the function keyword
  • the arrow function => is an implicit return. When you have one line of code return you do not need brackets or parentheses, but if its more than one line you do need it.

Big-O Complexity

Deals with the Time and Space Complexity, or rather how effective and efficient the code is. Often:

  • Look ups are faster O(1) Constant Time
  • Array deletion and shifting is more expensive O(n^2) Linear Time

Linked Lists

Singly Linked List Pros

  • Simplest thing to make compared to doubly.
  • Requires less memory, faster when deleting and insert
  • Good to use when memory and space is limited.

Singly Linked List Cons

  • Can be iterated or traversed on both sides (head & tail)
  • Requires less memory, faster when deleting and insert

Doubly Linked List Pros

  • Can be iterated or traversed on both sides (head & tail)
  • Requires less memory, faster when deleting and insert

Doubly Linked List Cons

  • Can be iterated or traversed on both sides (head & tail)
  • Requires less memory, faster when deleting and insert
